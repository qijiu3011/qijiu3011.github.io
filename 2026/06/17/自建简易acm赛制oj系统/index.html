自建 ACM 赛制 OJ 系统（Flask + Docker）

发表于 2026-06-17

基于 Flask 和 Docker 实现的轻量级ACM赛制在线评测系统，支持异步评测、多语言、可进行比赛与实时排行。

技术栈

  • 后端：Flask + Waitress（生产服务器）
  • 数据库：SQLite
  • 安全隔离：Docker（优先）+ psutil（本地回退）
  • 前端：原生 HTML + Ajax 轮询

核心实现

1. 异步评测队列

  • 提交接口只写入 submissions 表，状态 Pending，立即返回 submission_id
  • 后台常驻线程轮询 Pending 记录，逐个调用评测函数，更新状态
  • 前端每秒轮询 /result/ 获取结果，直到终态

2. 安全沙箱

  • 优先使用 Docker：动态创建 gcc:latest / python:3.10-slim 容器，限制内存 256MB、CPU 0.5 核
  • 无 Docker 时回退本地 psutil 进程树监控，2 秒超时、内存超限即杀灭进程组

3. 多题目与可见性

  • problems 表含 is_visible 字段，1 公开，0 仅比赛可见
  • 首页只显示公开题目，管理员可上传 ZIP 生成临时题目（is_visible=0），赛后一键发布

4. ACM 比赛系统

  • 表结构：contests（比赛信息）、contest_problems（多对多关联 + 显示顺序）
  • 提交记录增加 contest_id 字段，仅在比赛期间计入排名
  • 排名规则：AC 数量降序 → 总罚时升序
  • 罚时计算：(AC时间 - 比赛开始时间) / 60 + (AC前错误提交次数 × 20)

5. 管理员功能

  • 创建比赛：填写起止时间、从现有题库勾选题目、上传临时题目 ZIP（自动解析 desc.txt 和 input*.txt/output*.txt）
  • 赛后发布临时题目：自动迁移至 problems/ 并设为公开
