B - 小苯购物

问题描述

有 3 个优惠券，每个优惠券有两个参数 a、b：

  • 当当前商品价格 ≥ a 时，可以减少 b 元。

每个优惠券 最多使用一次，且可以选择 使用或不使用。

给定商品初始价格 n，求最终价格的 最小值。

  • 暴力枚举

使用 next_permutation 枚举所有排列，依次模拟使用优惠券的过程即可。

每次更新当前最小价格。

D - 小苯的序列染色

问题描述

给定一个长度为 n 的二进制字符串 t，初始时 全为 0。

可以进行任意次操作：

  • 选择任意 三个连续位置
  • 将其赋值为 "110"

给定目标字符串 s，问是否可以通过若干次操作将 t 变为 s。

考察知识点

  • 构造
  • 字符串性质
  • 贪心观察

题解思路

观察操作 "110" 的性质：

  1. 每次操作 最后一位一定是 0
    因此最终字符串 **最后一位不能是 1**。

  2. 去掉前导 0 后，剩余部分必须是 **连续的一段 1**。

如果第一个 1 后面出现 0，则无法通过操作构造出来。

满足上述条件即可构造成功。

C - 移动

问题描述

给定一个长度为 n 的字符串，由 . 和 # 组成：

  • # 表示障碍
  • . 表示可通行

现在有 q 次查询，每次给定两个位置 x 和 y，允许无视一次 任意长度的连续障碍，问：

要从 x 移动到 y，最少需要无视的障碍长度是多少？

考察知识点

  • 二分查找
  • 前缀/障碍位置记录
  • 贪心 + 查询优化

题解思路

暴力方法检查每次路径的障碍会超时。

优化做法：

  1. 记录所有障碍的位置到数组 a。
  2. 对于每次查询 [x, y]：
    • 使用 lower_bound 找到左端障碍
    • 使用 upper_bound 找到右端障碍
  3. 如果左右迭代器指向同一位置，说明区间内没有障碍，无需无视，答案为 0。
  4. 否则，计算 右障碍 - 左障碍 + 1 即为最少需要无视的连续障碍长度。

时间复杂度：

  • 预处理障碍：O(n)
  • 每次查询：O(log k)，k 为障碍总数
  • 总体：O(n + q log n)

E - 平衡排列

问题描述

定义一个排列 p 为 平衡排列，当且仅当存在一个索引 mid 使得：

$$
\sum_{i=1}^{mid} p_i = \sum_{i=mid+1}^{n} p_i
$$

现给定 n，请构造一个长度为 n 的平衡排列。

若不存在这样的排列，输出 -1。

考察知识点

  • 贪心构造
  • 数学性质
  • 子集和

题解思路

若要满足两侧和相等，则整个排列的总和必须为偶数。

排列 1..n 的总和为：

$$
S = \frac{n(n+1)}{2}
$$

若 S 为奇数，则无法分成两个相等的部分，直接输出 -1。

若 S 为偶数，则目标是构造一组数使其和为：

$$
\frac{S}{2}
$$

可以采用贪心策略：

从 n 到 1 依次遍历，如果当前数字加入左侧不会超过目标值，就放入左侧，否则放入右侧。

最终将左侧与右侧拼接即可得到满足条件的排列。
