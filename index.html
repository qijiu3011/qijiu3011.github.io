自建 ACM 赛制 OJ 系统（Flask + Docker）

发表于 2026-06-17

基于 Flask 和 Docker 实现的轻量级ACM赛制在线评测系统，支持异步评测、多语言、可进行比赛与实时排行。

技术栈

  • 后端：Flask + Waitress（生产服务器）
  • 数据库：SQLite
  • 安全隔离：Docker（优先）+ psutil（本地回退）
  • 前端：原生 HTML + Ajax 轮询

核心实现

1. 异步评测队列

  • 提交接口只写入 submissions 表，状态 Pending，立即返回 submission_id
  • 后台常驻线程轮询 Pending 记录，逐个调用评测函数，更新状态
  • 前端每秒轮询 /result/ 获取结果，直到终态


提交代码，选择题目和语言后提交。


评测完成后实时显示结果（AC/WA/TLE/MLE等）。

2. 安全沙箱

  • 优先使用 Docker：动态创建 gcc:latest / python:3.10-slim 容器，限制内存 256MB、CPU 0.5 核
  • 无 Docker 时回退本地 psutil 进程树监控，2 秒超时、内存超限即杀灭进程组

3. 多题目与可见性

  • problems 表含 is_visible 字段，1 公开，0 仅比赛可见
  • 首页只显示公开题目，管理员可上传 ZIP 生成临时题目（is_visible=0），赛后一键发布


题目列表，公开题目对所有人可见，比赛题目仅比赛期间可见。

4. ACM 比赛系统

  • 表结构：contests（比赛信息）、contest_problems（多对多关联 + 显示顺序）
  • 提交记录增加 contest_id 字段，仅在比赛期间计入排名
  • 排名规则：AC 数量降序 → 总罚时升序
  • 罚时计算：(AC时间 - 比赛开始时间) / 60 + (AC前错误提交次数 × 20)


比赛进行中的实时排行榜，按 AC 题数和罚时排序。

5. 管理员功能

  • 创建比赛：填写起止时间、从现有题库勾选题目、上传临时题目 ZIP（自动解析 desc.txt 和 input*.txt/output*.txt）
  • 赛后发布临时题目：自动迁移至 problems/ 并设为公开

项目地址

https://github.com/qijiu3011/oj

项目2 STL vector简易实现与解析

发表于 2026-04-04

在进行思考前，我们应先明确vector在STL中是什么分类。容器分为序列式和关联式，vector就是序列式的一种，与之同为序列式的有array、list、slist、deque等，实现较关联式容器的红黑树map等是更容易的。

正文：根据vector的源码，我复刻了一个简易版vector的大体框架

vector需要表示用户操作的当前数据的起始地址，结束地址，还需要真正的最大地址。所以共3个迭代器分别指向数据的头start，数据的尾finish，数组的尾endofstorage。由于只是一个初始的简易复现，对于尾使用了m_len表示数组长度，进行间接表示。

构造函数相关

在源码中，vector的初始化块进行了严谨的异常机制，由于知识过多以后再进行实现。

vector是一种class template，是在栈上分配的，所以我们不需要手动释放内存，生命周期结束后自动调用析构从而释放调用空间。但这不代表进程完全不进行delete操作，如上文框架中的m_Data指向的内存是在堆上通过 new[] 分配的，编译器不会自动释放，必须在 ~MyVector() 中写 delete[] m_Data，否则这块堆内存永远不会被释放，造成内存泄漏。

虽然不需要手动释放内存，但直接调用析构函数释放内存的操作是合法的

属性获取相关

下面主要是位置参数的获取，如vector的头和尾，返回末尾元素，返回当前size，元素容量以及是否为空等等。正常我们写end()返回的是finish，在简易实现部分我们返回start + pos

erase与clear

该函数用于清除指定位置的元素，删除一个范围内所有元素，实现就是将删除元素后的所有元素往前移动，对于vector是一个很大的开销，需要实现两类erase，分别为删除区间及删除单个元素。clear则等于erase(begin(), end())

insert

在vector插入元素分为三种情况

插入点后的现有元素多于新增元素个数

从pos点插入n个元素，先构造出start + pos - n大小空间，调整长度，将插入点 后的元素copy到备用空间，最后从插入元素位置填充新值

插入点后的现有元素少于等于新增元素个数

从pos点插入n个元素，先判断是否有备用空间，在插入点当前end()指针后构造一个元素，将原先插入点后的元素copy到备用空间，最后从插入点填充新值

如果备用空间不足

先配置新空间，再分段copy元素，填充插入值，最后调整迭代器位置

其中需要注意迭代器失效问题，因为插入元素可能使容器元素整体迁移导致存放原容器元素的空间不再有效，或是因为删除元素，使某些元素次序发生变化导致原本指向某元素的迭代器不再指向期望指向的元素。

五子棋游戏控制台与easyX版本

简介

使用 C++ 实现简单的的双人五子棋游戏，棋盘大小为 15×15。玩家轮流输入坐标进行落子，程序能够实时显示棋盘状态，并自动判断胜负和平局。

其中：

  • X 表示黑棋；
  • O 表示白棋；
  • 任意方向连续五个棋子即可获胜。

功能

1. 棋盘显示

使用二维数组 b[15][15] 存储棋盘状态，通过 print() 函数输出当前棋盘。

2. 落子与玩家切换

通过 check() 函数判断输入位置是否合法，并完成落子。利用变量 nowPl 记录当前玩家，每回合结束后切换到另一方继续游戏。

3. 胜负判断

checkWin() 函数以当前落子位置为中心，对横向、纵向以及两条对角线共四个方向进行搜索，当连续棋子数达到 5 时判定当前玩家获胜。

4. 平局判断

变量 cnt 用于统计落子总数，当棋盘被下满且无人获胜时，判定为平局。

主要函数

clear() 清屏
print() 输出棋盘
check() 判断落子是否合法
checkWin() 判断是否形成五连子
nowPlayer() 切换玩家
main() 控制整个游戏流程

图形界面版本

利用 EasyX 图形库实现了窗口版五子棋，通过鼠标完成落子，并实时绘制棋盘和棋子。程序支持胜负判断、平局判断、非法落子提示以及按 R 键重新开始游戏，提高了游戏的交互性和可视化效果。
在控制台版本的基础上，通过加入图形界面和鼠标事件处理，实现了更加直观的五子棋游戏，加深了对图形编程和事件驱动思想的理解。
